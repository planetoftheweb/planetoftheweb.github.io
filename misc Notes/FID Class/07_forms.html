Understanding Forms

  • Client & application communication
  • Respond to user input
  • Bridge between users and data

How forms work

  • Fill out form & Submit
  • Data sent to a processor
  • Processor performs an action
  • The browser receives a response

Processing Forms

The <form> tag

  • <form action="process.php" method="post" enctype="" >
  <label>Name: <input name="fullname"></label>
  <input type="submit" >
</form>
  • Action - Where to send the data to
  • Method - How to send the data
  • Enctype - How to encode the data
  • Try it Add a second last name field to this form.

The get method

  • <form method="get" action="processform.php"
  <label>Name: <input name="fullname"></label>
  <label>Title: <input name="title"></label>
  <input type="submit" >
</form>
  • get - Sends data through a URL
  • processform.php?fullname=Ray+Villalobos&title=Staff+Author
  • Name/Value pairs encoded into URLs

The post method

  • <form method="post" action="processform.php">
  <label>Name: <input name="fullname"></label>
  <label>Title: <input name="title"></label>
  <input type="submit" >
</form>
  • post - Sends data in the background
  • Used for longer forms
  • More secure

The label tag

  • 
<form method="post" action="processform.php">
  <label>Name: <input name="fullname"></label><br>

  <label for="title" >Title:</label>
  <input id="title" name="title"><br>
  <input type="submit" >
</form>
  • Ties label to the input field
  • Assumed if it wraps the input
  • Use FOR on label, ID on input
  • Try it Add another field with an extra label.
